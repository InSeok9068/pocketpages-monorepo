[vscode-pocketpages](tools/vscode-pocketpages/) 해당 LSP를 일단 Go, Typescript, Rust, Vue, Svlete 등의 메이저한 LSP랑 비교해서 좋은 구조를잘 가지고 있는지 실제 동작성에는 버그나 명확한 이슈가 있는지 꼼꼼히봐줘

1. 모든 코드를 전수 조사하면서 명확한 이슈나 버그가 있는지 찾기
2. 서로 연관되어있어서 함수를 공유하는 것들끼리 모아서 각각 서로 제공하는 함수 규약에 맞게 잘 정의되어서 동작하도록 되어있는지 검증
3. 실제 사용자가 경험할 수 있는정도의 버그나 이슈인지, 너무 작은 이슈로 고치는게 무의미한건 아닌지
4. 메이저한 LSP는 어떻게 이 이슈를 대응하고있는지
5. 마지막으로 캐싱의 버그가 숨어있는지, DX 관점에서 타이핑이나 자동완성에서의 동작 순서 및 캐싱 이런게 잘되어있는지

이와같은 5개의 고려사항을 잘 파악하면서 목표로르 가지고 하나씩 해당 관점을 다르게 보면서 이슈나 문제사항에 대한 보고서를 응답해줘
